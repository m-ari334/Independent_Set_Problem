Independent Set is NP-Complete

BCS 309 · Algorithms I · Spring 2026 · Mariam Hamouda

0

Problem Definition

▾

The Independent Set problem is defined as follows:

Input: An undirected graph G = (V, E) and a positive integer k.
Question: Does G contain an independent set of size ≥ k?
Independent Set: A subset S ⊆ V such that for all u, v ∈ S, the edge (u,v) ∉ E.

Intuitively, an independent set is a group of vertices where no two are connected by an edge. Applications include wireless channel assignment (non-interfering transmitters), register allocation in compilers, and social network analysis (finding non-conflicting groups).

💡 Key Complement Relationship: S is an independent set in G if and only if V \ S is a vertex cover of G. This direct complement relationship is the heart of the NP-completeness proof.
1

Step 1 — Independent Set ∈ NP

▾

To show a problem is in NP, we must exhibit a polynomial-time verifier — an algorithm that, given an instance and a certificate (proposed solution), checks in polynomial time whether the certificate is valid.

Certificate: A subset S ⊆ V (the claimed independent set).

Verifier Algorithm:
1. Check |S| ≥ k — takes O(|S|) time.
2. For every pair (u, v) ∈ S × S, check that (u,v) ∉ E — takes O(|S|²) time.
3. Accept if both checks pass; reject otherwise.

The verifier runs in O(|V|²) time — polynomial in the size of the input. Therefore, Independent Set ∈ NP. ∎

2

Step 2 — NP-Hardness: Vertex Cover ≤ₚ Independent Set

▾

To show NP-hardness, we reduce from Vertex Cover (already known to be NP-complete) to Independent Set in polynomial time.

Vertex Cover: Given graph G = (V, E) and integer k, does G have a vertex cover of size ≤ k?
(A vertex cover C ⊆ V covers every edge — for every (u,v) ∈ E, at least one of u, v is in C.)

Reduction Construction:

Given a Vertex Cover instance (G, k), construct the Independent Set instance:

f(G, k) = (G, |V| − k)

The graph G is unchanged. Only the target size changes from k to |V| − k.

Correctness Proof — Both Directions:

⟹ Forward Direction

Suppose G has a vertex cover C of size ≤ k. Let S = V \ C. Then |S| ≥ |V| − k.

Claim: S is an independent set. Proof by contradiction: if u, v ∈ S are adjacent (i.e., (u,v) ∈ E), then neither u nor v is in C. But then edge (u,v) is not covered — contradiction with C being a vertex cover. ∴ S is independent. ✓

⟸ Backward Direction

Suppose G has an independent set S of size ≥ |V| − k. Let C = V \ S. Then |C| ≤ k.

Claim: C is a vertex cover. For any edge (u,v) ∈ E: since S is independent, at least one of u, v ∉ S, so at least one of u, v ∈ C. ∴ every edge is covered. ✓

Polynomial Time: The construction f maps (G, k) → (G, |V|−k). This requires only computing |V|−k — O(1) arithmetic — and passing G unchanged. Total time: O(1). ∎
✓

Conclusion

▾

We have shown:

1. Independent Set ∈ NP (polynomial-time verifier exists).
2. Independent Set is NP-hard (Vertex Cover ≤ₚ Independent Set via complement reduction).

∴ Independent Set is NP-Complete. ∎

This means: (a) there is no known polynomial-time algorithm for Independent Set, (b) if anyone found one, it would solve ALL NP problems in polynomial time (P = NP), and (c) the greedy heuristic we use trades optimality for tractability on large graphs.

Reference: Cormen et al., Introduction to Algorithms, 4th ed., Chapter 34 (NP-Completeness).

Step 1: Showing IS ∈ NP (1 min)

  • Certificate: the subset S itself (polynomial-sized)
  • Verifier: check |S|≥k, then check all pairs in S×S — O(n²) time
  • Conclusion: a polynomial-time verifier exists ⟹ IS ∈ NP

Step 2: Reduction from Vertex Cover (3 min)

  • State Vertex Cover is NP-complete (from CLRS Chapter 34)
  • Construction: map (G, k) → (G, |V|−k) — same graph, complement target
  • Forward direction: if C is a vertex cover of size ≤k, then S=V\C is an IS of size ≥|V|−k
  • Backward direction: if S is an IS of size ≥|V|−k, then C=V\S is a vertex cover of size ≤k
  • Polynomial time: the construction runs in O(1) — trivial mapping
Anticipated Q: "Why can we use Vertex Cover as the source problem?"
A: Vertex Cover is already proven NP-complete (CLRS Theorem 34.5). To prove a new problem NP-complete, we need ONE known NP-complete problem to reduce from. We chose Vertex Cover because the complement relationship makes the reduction O(1) — the simplest possible construction.

PART 3 Algorithm Design 5–8 min

Paradigm: Greedy (2 min)

  • The greedy paradigm: make locally optimal choices at each step without backtracking
  • Our rule: always pick the vertex with the MINIMUM degree
  • Intuition: a low-degree vertex has fewer neighbors, so adding it to S eliminates fewer future candidates — greedy "preserves options"

Complexity Analysis (2 min)

  • Each iteration: finding min-degree vertex = O(n), removing it + neighbors = O(n)
  • At most n iterations → total: O(n²)
  • Contrast: brute force = O(2ⁿ · n²) — exponential
  • Trade-off: greedy is fast but not guaranteed optimal

Approximation Analysis & Counterexample (2 min)

  • Greedy has no constant approximation guarantee for Maximum IS in general graphs
  • Show the Petersen graph counterexample: greedy returns 3, optimal is 4
  • Why we still use it: practically fast and often near-optimal on sparse graphs
  • Alternatives: simulated annealing, genetic algorithms (mention only)
Anticipated Q: "Why did you choose min-degree over max-degree greedy?"
A: Min-degree greedy preserves more candidates for the IS. When you add a low-degree vertex, you eliminate only a small neighborhood. Max-degree greedy would eliminate many vertices at once, shrinking the remaining graph faster — but this hurts the IS size. Empirically, min-degree gives larger ISs on random graphs.
Anticipated Q: "Is there a polynomial-time exact algorithm for IS?"
A: No known polynomial-time exact algorithm exists for general graphs — this is precisely because IS is NP-complete. Exact algorithms (like our brute force) are exponential. Special graph families (trees, bipartite graphs, perfect graphs) do admit polynomial-time exact algorithms, but general graphs do not unless P=NP.
PART 4 Q&A Preparation 5 min

Core Concepts to Master

  • What is NP? — decision problems verifiable in polynomial time
  • What is NP-hard? — every NP problem reduces to it in poly-time
  • Why does the complement relation IS↔VC work as a reduction?
  • Why is both forward AND backward direction needed in a reduction?
  • What does "polynomial-time reduction" mean precisely?
  • Difference between approximation algorithm and heuristic

Key Takeaway

The core insight: IS and Vertex Cover are complementary views of the same structure. This makes the reduction trivial — no transformation of the graph is needed, only the target size changes. This elegance is rare and worth emphasizing.

Problem: Independent Set

Given an undirected graph G=(V,E) and integer k, does G contain an independent set of size ≥ k? This is a canonical NP-complete problem with applications in wireless network channel assignment, compiler register allocation, and social network analysis.

Algorithms Implemented

Greedy Heuristic — O(n²): At each step, select the minimum-degree vertex, add it to the independent set, and remove it and all neighbors. Fast and practical but not guaranteed optimal.

Brute Force Exact — O(2ⁿ · n²): Enumerate all 2ⁿ subsets, check each for independence, return the largest valid subset. Optimal but exponential — feasible only for n ≤ 20 or so.
